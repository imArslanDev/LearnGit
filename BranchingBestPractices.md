### 1 One Feature = One Branch 
Create a dedicated branch for each feature, bug fix, orimprovement.

### 2 Never Develop Directly on Main Branch
Protec the main branch and keep it stable.
All changes should go through pull requests.

### 3 Delete Merged Branches
Once a branch is merged, delete it to keep the repository clean.

### 4 Use Meaningful Branch Names
Use clear, descriptive names that indicate the purpose of the branch.

### 5 Keep Branched Short-Lived
Merge your canges as soon as possible to reduce the risk and conflicts.
